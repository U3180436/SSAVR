Quality Attribute Scenarios

What are Quality Attribute Scenarios in Software Architecture?

Quality Attribute Scenarios (QAS) in Software Architecture are descriptions of how a software system should behave with respect to specific Quality Attributes! Some exaples of whcih may be reliability, performance, security and many more.

Each QAS are created by a development team to help software architects and developers define the why and how the requirements for a system should be implemented and how they should function! (Kazman et al. 2006)

What is the structure of a Quality Attribute Scenario?

A Quality Attribute Scenario consists of several parts:

  1. Stimulus: An event or action that triggers the scenario. For example, the stimulus could be a user entering data into a form, or a system receiving a large number of requests.
  2. Source: The entity that generates the stimulus. It could be a user, another system, or even a physical device.
  3. Environment: The context in which the stimulus occurs. It includes factors such as the hardware and software platform, network conditions, and user workload.
  4. Artifact: The component or subsystem of the software system that is being evaluated. For example, it could be the database, the user interface, or the network protocol.
  5. Response: The expected behavior of the system in response to the stimulus. For example, the response could be that the system processes the request within a certain time limit, or that it denies access to unauthorized users.
